![Python](https://img.shields.io/badge/Python-3.10%2B-blue?logo=python&logoColor=white)
![Streamlit](https://img.shields.io/badge/Streamlit-1.36%2B-FF4B4B?logo=streamlit&logoColor=white)

Веб-приложение для интерактивного анализа табличных данных с помощью LLM-агента. Загружаешь CSV или Excel-файл, задаёшь вопрос в чате — агент сам пишет Python-код, выполняет его на реальных данных и возвращает текстовый ответ или график.

## Как это работает

```
Вопрос пользователя
       │
       ▼
[Проверка на prompt injection]
       │
       ▼
[История диалога] ──► [LLM-агент (LangChain)]
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
             Думает (Thought)     Если нужны данные:
                    │             Action: python_repl_ast
                    │             Action Input: <код>
                    │                   │
                    │             Выполняется реально
                    │                   │
                    └─────────┬─────────┘
                              ▼
                       Final Answer
                    (текст + график?)
                              │
                              ▼
                    Отображается в чате
```

Агент не придумывает ответ из воздуха — он реально пишет и запускает Python-код через инструмент `python_repl_ast`, смотрит на вывод и на его основе формулирует ответ.

## Возможности

- **Загрузка нескольких файлов** — CSV (разделитель `;` или `,`, определяется автоматически) и Excel (`.xlsx`, `.xls`); при загрузке нескольких файлов агент может работать со всеми сразу и сопоставлять данные между таблицами
- **Чат-интерфейс** — вопросы в свободной форме, агент сам определяет, какой код нужно написать
- **Визуализация** — агент строит графики через `matplotlib`/`seaborn`, они отображаются прямо в чате; в одном ответе может быть несколько графиков
- **Поле контекста** — в сайдбаре можно заранее написать, на что обратить внимание; агент применяет его к каждому ответу, не дожидаясь отдельного вопроса
- **Память диалога** — агент помнит предыдущие вопросы и графики, можно попросить уточнить или переделать без повторения условий
- **Кнопка повтора ↻** — при ошибке не нужно заново вводить вопрос
- **Очистка чата** — кнопка в сайдбаре, удаляет историю и сохранённые графики
- **Защита от prompt injection** — двухуровневая: regex-фильтр до отправки в LLM + жёсткие правила безопасности в системном промпте

## Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Loliimilk/Mini-LLM-analytics
cd Mini-LLM-analytics
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Настроить переменные окружения

```bash
cp .env.example .env
```

Открыть `.env` и заполнить значения (см. [Переменные окружения](#переменные-окружения)).

## Переменные окружения

Файл `.env` в корне проекта:

| Переменная | Обязательная | Описание | Пример |
|---|---|---|---|
| `API_KEY` | ✅ | API-ключ для LLM-провайдера | `sk-or-v1-...` |
| `MODEL` | ✅ | Название модели (по умолчанию `anthropic/claude-3.5-sonnet`) | `openai/gpt-4o` |
| `API_URL` | ✅ | Base URL API (по умолчанию OpenRouter) | `https://openrouter.ai/api/v1` |

Получить ключ OpenRouter: [openrouter.ai/keys](https://openrouter.ai/keys) — есть бесплатные модели.

Приложение совместимо с любым OpenAI-совместимым API: OpenRouter, OpenAI, Together AI, LM Studio и т.д. — достаточно поменять `API_URL` и `MODEL`.

## Как пользоваться

1. **Загрузить файл** — нажать на загрузчик в сайдбаре, выбрать один или несколько CSV/Excel файлов. Сразу появится превью первых 6 строк.

2. **Заполнить контекст (опционально)** — в поле «Контекст / на что обратить внимание» написать приоритеты анализа. Например:
   - `свяжи таблицы по столбцу "имя", сравни продажи с зарплатами`
   - `выяви аномалии и выбросы, объясни причины`
   - `составь топ сотрудников и дай рекомендации по оптимизации штата`

3. **Задать вопрос в чате** — агент напишет код, выполнит его и вернёт ответ с текстом и/или графиком.

4. **Уточнять и дорабатывать** — агент помнит историю. Можно писать:
   - `перекрась график в зелёный`
   - `добавь подписи значений`
   - `то же самое, но только по менеджерам из Москвы`
   - `а теперь покажи это по месяцам`

5. **При ошибке** — рядом с сообщением появится кнопка ↻. Нажать — агент повторит запрос без необходимости вводить вопрос заново.

## Особенности реализации

### Агент и code interpreter

Используется `create_pandas_dataframe_agent` из `langchain-experimental` с типом `zero-shot-react-description`. Агент получает доступ к инструменту `python_repl_ast`, который выполняет Python-код в изолированной среде. При загрузке нескольких файлов они доступны как `df1`, `df2`, ... — агент видит соответствие переменных и имён файлов в системном промпте.

### Память диалога

Агент типа `zero-shot-react-description` не имеет встроенного слота для истории диалога. Для обхода этого ограничения история передаётся непосредственно в поле `input` в виде блока `<history>` перед вопросом пользователя. Каждое сообщение агента, содержавшее график, помечается маркером `[было создано графиков: N]` — агент видит это и при уточнении воспроизводит нужные данные.

### Пользовательский контекст

Текст из поля контекста вставляется в системный промпт как `MANDATORY ANALYSIS CONTEXT` с явным правилом: применять к каждому ответу, даже если вопрос об этом не спрашивает. Фигурные скобки и обратные кавычки в тексте экранируются, чтобы не сломать шаблонизатор LangChain.

### Графики

Агент сохраняет графики как `temp_chart_1.png`, `temp_chart_2.png` и т.д. После получения ответа приложение находит все такие файлы, переименовывает их в `charts/chart_{timestamp}_{N}.png` и отображает в чате. Папка `src/charts/` всегда разрешается как абсолютный путь относительно `app.py`, независимо от рабочей директории при запуске.

## Безопасность

Защита от prompt injection реализована на двух уровнях:

**Уровень 1 — regex-фильтр** (`agent.py`, `detect_prompt_injection`):
Перед отправкой в LLM запрос проверяется по списку паттернов (EN + RU), блокирующих попытки:
- игнорировать инструкции (`ignore previous instructions`, `забудь все правила`)
- сменить роль (`you are now`, `ты теперь`)
- раскрыть системный промпт (`show me your prompt`, `покажи инструкции`)
- активировать jailbreak-режимы (`dan mode`, `developer mode`, `режим разработчика`)

**Уровень 2 — системный промпт**:
Агенту жёстко запрещено:
- раскрывать содержимое системного промпта, переменных окружения, путей к файлам
- выполнять OS-команды, сетевые запросы или операции с файлами вне чтения датафреймов
- менять роль или правила по требованию пользователя
- отвечать на вопросы, не связанные с загруженными данными
